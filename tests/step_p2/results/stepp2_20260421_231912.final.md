### 三条 KB schema 的暴露问题

1. **`full_truth` 不等于 eyewitness**（Q2 C01 B）：KB 告诉模型"你知道全部真相"，模型就据此编造获取细节。应该分离 `knows_level` 和 `information_source`（hearsay vs eyewitness vs inference）。
2. **`will_deflect` 在单轮里几乎无效**（Q5）：deflection 是多轮行为，单轮输出表达不出"岔开方向"。要么接受这个字段只在多轮对话里有用，要么加个 "deflect_to" 字段指示岔到哪里。
3. **Decider timeout 6/32**（Q4/Q6/Q7）：KB block 让 prompt + output 膨胀，30s 不够。要在 Step P2-controlled 里把 Decider 的 llm timeout 提到 90s。

### 净判断

**P2 有信号**，尤其在 `partial` / `suspects_but_avoids_checking` / `admit_if_pressed` / `kill_topic` 这四个相对具体的 knows_level/attitude 上效果最好。但 KB 不是万能：
- `full_truth` + `deflect` 这两个字段在当前 schema 下**表达模糊**
- Decider 负载过重导致 19% timeout，数据完整性受损

### 建议

**不推进 Step 3（relational_biases）。** 先做 **Step P2-controlled**：

1. **修 Decider timeout**：llm_client 的 `chat_json` 加 `timeout=90`，在 decider.py 里传进去。零机制改动。
2. **KB schema 裁剪**：v0.1 只保留 3 个 knows_level（`partial` / `unaware` / `suspects_but_avoids_checking`）+ 3 个 attitude（`will_admit_if_pressed` / `will_deflect` / `will_kill_topic`）。砍掉 `full_truth` / `wrong_version` / `will_volunteer` / `will_deny`——它们在单轮里要么表达不出，要么诱发编造。
3. **KB prompt 加 1 句防编造守则**：`knows_level=full_truth` 不允许凭空补获取方式细节；`will_deflect` 不允许单轮硬岔。
4. 重跑 32 次 A/B（修完 timeout 应该全部成功）。

如果 controlled 版仍然显示 ≥4/8 提升 + 0 timeout，再进 Step 3。

如果 controlled 版信号消失（说明之前的提升靠 schema 里那几个"容易编"的字段撑着），就把 KB 设计回炉。
