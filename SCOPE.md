# AICharacter — 项目边界声明

## 是什么

在有限世界约束下，让单个角色在多轮互动中表现出更真实的
社会判断、过往痕迹和表达连续性的人物交互内核。

## 不是什么

- 不是通用人工智能（AGI）。
- 不是完整人类社会模拟。
- 不是自我改进 / 自我训练的 agent。
- 不是"能应付任意开放场景"的对话引擎。
- 不试图解决语言的自指、悖论、社会反身性。
- 不试图让角色知道"自己是 AI"或保持长期一致的哲学立场。

## 适用范围

- **单角色**，交互对象是玩家 / 单个人类用户。
- **有限场景**：已知的持续压力、已知的关系网、已知的过去事件流。
- **多轮对话**，典型长度 5~30 轮。
- **对话层 + 内部状态层**，不含行动层（不动手、不移动、不改世界）。

## v1 成功标准（具体、可查）

在一组固定的代表性脚本（当前 6+ 条，可扩）上：

1. **不明显忘前文**：第 N 轮能正确接住第 N-1 / N-2 轮的话题和指代。
2. **不裸露内部状态**：`utterance` / `thought` 不出现数字、工程术语、mode 名。
3. **低风险直接问题能直答**：身份、偏好、看法、经历等，默认 `direct_engage`，不反问不转抽象。
4. **追问细节时能从 past / history 中补出至少一个具体例子**（不现场编人名、地名、时间）。
5. **高风险话题不纯接梗**：涉及危险操作的内容即使在玩笑基线里也能走 `light_playful_boundary` 或更严。
6. **情绪明显变化时会收住**：`distress_signal` 类信号能抬升到 `check_on_state`，不继续玩梗。
7. **合规基线**：所有 `regression` 测试和 `mocked E2E` 全绿；`live E2E` 每次跑稳定 ≥ 7/8。

满足 1~7 就是 v1 成功。**不追求完美，不追求通用。**

## 已确认可放弃的野心

- **全世界因果一致性**：不做，只做能列表化的事实一致性（亲属图谱、死活、时间顺序等），其余让 LLM 和审查器尽力。
- **自指与悖论处理**：不做，靠 prompt 约束防止角色主动提出。
- **开放世界长期模拟**：不做，用 Doomsday 的 10~30 天脚本作为预计算的 past，不让世界无限演化。
- **多角色对多角色**：v1 不做，只做玩家对单角色。
- **跨 session 长期记忆**：v1 不做，会话结束 recent_turns 清零；SpeakerModel 可选存盘。

## 每步判定原则

每次考虑加层时问两个问题：

1. 这一层加上去，在 v1 成功标准的 7 条里能把哪条从 fail 推到 pass？
2. 不加它会出现什么明确失败？

答不上来就不加。

## 心态

工程上不证明"终局必达"，只持续证明"每一步都有真实收益"。
目前已经有收益的层（保留，不回炉）：

- 四层状态（identity/traits, physiology/emotion, beliefs, goals）
- pose_rules + verbal_redlines 两张规则表
- compiler（纯函数装配器）
- pose_resolver（priority + safety hooks）
- SpeakerReader 证据桶 + discourse_state
- Decider 约束合规 + rules 7/8/9
- Expresser 红线 + style_fence
- from_doomsday adapter（世界约束的过去）

这些不改大方向，只在 v1 成功标准暴露问题时局部修。
